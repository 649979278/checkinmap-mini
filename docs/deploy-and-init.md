# 北京地图打卡小程序部署与初始化

## 1. 配置小程序云环境
- 打开微信开发者工具，载入本项目。
- 确认 `miniprogram/app.js` 中环境 ID 为 `cloud1-4gn7jrqa537ee147`。
- 点击右上角“云开发”，确认当前环境也是 `cloud1-4gn7jrqa537ee147`。

## 2. 上传并部署云函数
- 在微信开发者工具左侧 `cloudfunctions` 目录中，依次右键以下目录：
  - `seedScenicSpots`
  - `getHomeMapData`
  - `searchPlaces`
  - `upsertCheckin`
  - `getMyCheckins`
- 每个目录选择“上传并部署：云端安装依赖”。
- 原来的 `quickstartFunctions` 可以保留，但首页已不再依赖它。

## 3. 初始化数据库
- 打开 `cloudfunctions/seedScenicSpots`。
- 右键选择“本地调试”或在云开发控制台中打开云函数测试面板。
- 直接执行一次 `seedScenicSpots`，不需要传入参数。
- 成功后会自动写入 `scenic_spots` 集合。
- `user_checkins` 不需要手工初始化，用户首次打卡会自动创建。

## 4. 手工检查数据库
- 打开微信开发者工具右上角“云开发”。
- 进入数据库。
- 检查是否存在：
  - `scenic_spots`
  - `user_checkins`
- `scenic_spots` 应能看到北京城六区的内置景点数据。

## 5. 真机联调建议
- 先执行一次 `seedScenicSpots`。
- 返回首页，确认地图出现景点 marker 和六区进度。
- 选择任一景点打卡，确认 marker 变为绿色对号。
- 搜索一家饭店并打卡，确认它出现在“我的打卡”页，但不影响区级进度。

## 6. 上线流程
- 在微信开发者工具点击“上传”。
- 填写版本号和版本备注。
- 登录微信公众平台，进入版本管理。
- 选择刚上传的开发版本，提交审核。
- 审核页首页路径填写：`pages/index/index`。
- 审核通过后点击“发布”。

## 7. 提审前检查
- 小程序名称、简介、图标、服务类目已配置。
- 隐私说明已补充：
  - 使用 `openid` 区分用户
  - 存储用户填写的打卡备注
  - 使用云开发数据库保存打卡数据
- 首页、搜索、打卡、我的打卡四条主链路已真机验证。

## 8. 重置景点数据
- 如果只想重新灌入景点数据，优先再次执行 `seedScenicSpots`。
- 该云函数是幂等的，重复执行会更新同名同区景点，不会重复新增。
- 如果要彻底清空 `scenic_spots` 后重灌，请先在云开发控制台手动删除集合数据。
- 删除数据属于危险操作，执行前请先确认是否保留现有数据。
